# 经营建造与防守游戏原型 需求文档 (Base Builder & Defense)

## 一、 核心玩法概述 (Core Gameplay Loop)

这是一款结合了**基地建设 (Base Building)** 与 **基地防守 (Tower Defense / Siege Survival)** 的部分系统的原型框架。
玩家的主要目标是在安全的时段收集资源并建设基地，然后在危机来临时利用自己建造的防御体系和工事抵御敌人的轮番进攻，保护核心建筑（基地核心）不被摧毁。

* **发育（建造）期**：规划营地、建造资源产出建筑、布置防御工事。
* **入侵（防守）期**：由四面八方涌来的敌人，考验资源的合理规划与阵型防御强度。

## 二、 核心子系统拆解 (Core Systems Breakdown)

### 1. 资源系统 (Resource System)

* **基础代币**：例如“金币”、“木材”、“能源核心”等。
* **获取方式**：
  * 建筑物随时间阶段性产出。
  * 击败敌人掉落。
* **代码设计**：需要一个统一的全局大管家 `ResourceManager`，负责资源的增减计算与数据广播（供 UI 刷新）。

### 2. 建造与网格系统 (Building & Grid System)

* **建造行为**：在指定的网格（Grid）上预览并放置建筑模型。
* **建筑功能分类**：
  * **主基地 (Core/Nexus)**：胜负关键，被摧毁则判定游戏失败。
  * **资源建筑 (Economy)**：持续产出对应资源的单位。
  * **防御设施 (Turrets)**：如单发弩箭塔、AOE能量塔等、具有不同的射程范围和伤害逻辑。
  * **阻挡工事 (Barriers)**：如城墙、拒马，没有攻击力但有高额血量，用来卡位和更改敌怪路线。
* **代码设计**：`BuildingManager` 控制玩家输入、合法放置检测（碰撞检测、花费检测）。

### 3. 波次与敌人系统 (Wave & Enemy System)

* **敌人行为 (Enemy AI)**：依赖 Unity NavMesh 系统。敌人出生后将设定目标为主基地，如若遇到城墙死路，将会攻击最近的城墙来开路。
* **波次发生器 (WaveManager)**：配合数据配置（ScriptableObject），决定第N波在哪几个刷怪点、产出多少只什么类型的怪物。
* **怪物分类推演**：普通近战（炮灰）、高防肉盾（吸引塔的火力）、远程怪（隔墙输出抛射物）、爆破怪（专门炸墙）。

### 4. 战斗与伤害框架 (Combat & Damage Framework)

* **统一接口**：为所有的建筑、怪物抽象出 `IDamageable` 接口，赋予它们 HP 属性与受伤/死亡事件。
* **战斗判定**：塔进行半径扫描（SphereOverlap 搜寻）、发射弹道 / 射线击中目标产生伤害。

## 三、 目录结构规划建议 (Directory Structure)

为保证项目整洁且遵循单一职责，已在本路径下开辟相应子级目录：

* `Art/`：用于存放所有相关的 3D 模型、贴图和材质 (Materials)。
* `Data/`：存放各种 ScriptableObject 配置文件（防御塔数据、波次配置数据等）。
* `Docs/`：存放需求开发与策划文档（如您正在阅读的这篇文档）。
* `Prefabs/`：存放提前配置好的各类预置体资源（建筑Prefab、怪兽Prefab、弹道/特效Prefab）。
* `Scenes/`：测试与正式运行游戏的 Scene 文件。
* `Scripts/`：本系统所有的 C# 代码脚本，后期可再划分为 Entities, Core, Managers 等子目录。

## 四、 AI 托管与自验证系统 (AI Self-Hosting & Verification)

为了实现“纯AI托管”，本项目集成了自验证闭环：

### 1. 增强型日志系统 (Log System)

* **功能**：所有核心行为（建造、攻击、资源变动、寻路）必须记录日志。
* **开关控制**：通过 `GameConfig` 或全局静态变量控制输出等级，平衡调试需求与性能开销。

### 2. AI 验证接口 (AI Verification Hub)

* **接口原则**：精简、语义清晰、通过参数扩展。
* **用途**：允许 AI 通过代码/控制台直接查询当前资源总量、场上怪物数量、特定建筑坐标等，用于判断逻辑是否正确。

### 3. 开发闭环流 (The Loop)

* **编写 -> 运行(日志) -> 数据抓取(验证) -> 结果判定 -> 修正/出报告**。

### 4. 人工审核报告 (Human Review Report)

* AI 必须整理包含以下内容的 `.md` 报告：
  * **关键代码实现**。
  * **自验证逻辑描述**（我如何确认它是对的）。
  * **实时数据支撑**（验证时的截图/录屏或关键数据截断）。

## 五、 开发步骤 Roadmap

1. **第一阶段：底层设施**
    * 建立基础大框架，如 `ResourceManager` 基础运转，`BuildingManager` 空地落房。
2. **第二阶段：最简战斗闭环**
    * 制作第一种怪物能通过 NavMesh 跑向主基地。
    * 制作第一把防御塔能开火打死怪物。
3. **第三阶段：加入游戏流**
    * 完成“建塔 -> 资源扣除 -> 倒计时出怪 -> 打怪得分获资源”的循环流 `GameMode`。
4. **第四阶段：拓展深度**
    * 城墙与动态 NavMesh 障碍更新，敌人应对策略。
    * 加入数据配置，丰富塔和敌人的品类。
